Beat 3 · What bw is (developer view)

What bw actually is — the SWE version.

You probably want this in fewer words. Here's the structural shape.

Lives where your code lives.

  • Storage: orphan branch in a git repo — plain JSON files. Never merges into your code branches; never pollutes your tree.
  • Install: single Go binary. One-line curl installer or build from source. No daemon, no service, no SaaS.
  • Cross-AI portable: anything that can shell out to bw show reads the same store — Claude, Gemini, OpenAI, Cursor, your own tooling.
  • License: MIT, jallum's project at github.com/jallum/beadwork.
  • Survives compaction. Tickets and comments persist across session boundaries, agent restarts, and machine swaps. The killer feature.

The three layers, in your terms

  • One agent, across time: persistent context for one Claude/Gemini session graph — bw show on session start replaces the "where were we?" preamble.
  • Many agents, one human: async pipelines — a security-review agent and a style-review agent both write structured verdicts to the same PR ticket; a synthesizer agent reads them.
  • Many humans, many agents: push the orphan branch to a shared private remote. Other team members' agents read and write to the same store. The branch is just git — standard PR/review/audit workflows apply.

Trust story is structural, not marketing. Lives in your repo. Free. Cross-AI. Open source. Survives compaction. No SaaS in the path.
