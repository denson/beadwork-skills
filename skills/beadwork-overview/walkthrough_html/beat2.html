Beat 2 · What bw is

Five structural things make bw different.

Most "AI memory" tools today are SaaS — you sign up for an account, your AI's notes live on someone else's server, and you pay for the privilege. bw is structurally different.

Lives where your code lives.

  1. It lives in your repo. Your disk, your git, your remote of choice. Not on jallum's servers. Not on any vendor's servers. The data goes wherever your code goes — and stops wherever you stop pushing.
  2. It's free. No signup, no quotas, no per-user pricing. Single binary, install with one curl command. The price is "you already have git."
  3. It's cross-AI portable. Claude, Gemini, OpenAI, Cursor, anything that can shell out to bw show reads the same store. Your AI's memory is portable across vendors — if Anthropic raises prices and you switch to Gemini, your agent memory comes with you.
  4. It's open source. jallum's project, MIT-licensed. The format is plain JSON files in a git branch. If bw disappeared tomorrow, your data is still there.
  5. It survives compaction. The killer feature. Tickets and comments persist across session boundaries, agent restarts, machine swaps, and the years.

This combination — durable, portable, free, open, in your repo — is what makes bw different from "AI memory" SaaS tools. The trust story is structural, not a marketing claim.
