beadwork-for-meta-analysis · Beat 2 · How meta-analysis works in bw

Read the work, not the chat.

A meta-analysis agent doesn't watch other agents work. It reads what they wrote afterward.

Read the work, not the chat.

  • bw show <ticket> returns the full thread for one ticket — title, description, every comment in order, every author, every timestamp.
  • bw list returns ranges of tickets matching filters (status, type, label, date range).
  • bw history <id> returns the diff history of edits, so you see how a ticket evolved, not just its current state.

The audit agent feeds those into its prompt and asks questions a single agent can't answer alone:

  • "Did agents A and B apply the same criteria across these 12 reviews?"
  • "Is there drift on naming convention over the last quarter?"
  • "Agent C said X; agent D's comment two days later contradicts it — flag for human review."

Same git-backed substrate as memory and the bus. New analysis layer on top.
