beadwork-as-bus · Beat 2 · The mechanism

Conflict-free by construction.

Multiple agents read and write the same bw store. Each comment records who wrote it (a role name, an agent name, a timestamp) — so when Agent B reads what Agent A wrote yesterday, B knows who said it and when.

Different tickets, different files. No collision.

Two agents working on different tickets simultaneously never touch the same bytes — bw stores each ticket as a separate file under the orphan branch. No merge drivers, no lock files, no coordination overhead. Two agents writing different files at the same time just works.

The substrate is just shell. bw exposes a few simple commands (bw show, bw comment, bw close). Any agent that can shell out participates the same way — whether it's a different session of the same model or, in rarer cases, an agent from a different lab altogether.
