Beat 1 · The hook

A team of specialist agents, sharing one workspace.

If your AI is one agent in one chat, memory is the whole story. The moment you want more than one agent working on something — a design agent, an implementation agent, a review agent — memory alone isn't enough. They need to talk to each other in a way that survives.

Most of the time these agents are sessions of the same model from the same lab — you're running multiple Claude sessions, or multiple Gemini sessions, each playing a specialist role. The substrate works the same way across labs too, but day-to-day it's usually one lab, many sessions.

Many agents. One ticket. Asynchronous, signed, durable.

bw (beadwork) is the channel. Multiple agents read and write the same ticket store. Each comment is signed (the agent's role + name + timestamp). No two agents step on each other's writes.

The interaction stays the same: you talk to your agent. What's new is that your agent is now part of a small team that talks through bw.
