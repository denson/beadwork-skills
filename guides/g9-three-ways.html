beadwork · for developers

The same tool, three ways.

Beadwork is a command-line tool, so a coder can drive it by hand. Put an agent on top and the same things get easier and more powerful — and that's exactly the path a non-coder takes, too: they just talk. Here are the cooler things it does, across all three.

What you're doing Coder, by hand Coder, with an agent Non-coder, with an agent
Log a decision bw create the ticket, bw comment the options and reasoning, bw close with the verdict. "We're going with X over Y because Z" — it writes the ticket, the reasoning, and the verdict as you work. Tell your assistant the decision in plain words; it's recorded, with the date.
See what's next Wire dependencies with bw dep add, then bw ready lists what's unblocked. It keeps the dependency map and tells you the next thing to pick up — and you can still run bw ready yourself anytime to check. "What can I actually start today?" — answered from the plan.
Catch up after time away bw recap --all — a cross-repo digest of what changed. "What moved while I was out?" — it summarizes and flags what needs you. "What happened on my projects this week?" — a plain summary.
Get the "why" back bw show / bw history — the full thread and how it evolved. "Why did we land on this?" — it reconstructs the thread and explains it. "How did we decide that?" — the story, in plain language.
Run a team of specialists bw sync to share the store; coordinate through tickets yourself. Design / implement / review agents hand off through tickets — a pipeline that runs while you're away. Your assistant and a teammate's assistant coordinate on a shared item — no meeting.
Audit the work Read the tickets, or bw export and slice the data yourself. An agent reads the whole history — finds contradictions, checks methodology, writes the summary. "Did anything we decided contradict itself?" — it checks and tells you.

If you write code, you can already do the left column by hand — and beadwork makes that flow more efficient even before an agent gets involved. The middle and right columns are where it really earns its place — the agent doing the bookkeeping, and the same power available for all the non-code work your tools never touched. Most coders who try it there keep it for exactly that.

bw (beadwork) is an open-source tool by jallum. This guide is part of a plain-language set for using it without writing code.
